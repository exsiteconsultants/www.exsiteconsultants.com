Front End Developer - Department for International Trade

Apr 2016 - Current

I have worked as a Front-end/full-stack developer on an online service to replace an existing Microsoft Dynamics CRM platform. The new platform follows the GDS guidelines for style and layout adapted to deliver a CRM. Development is agile and user-driven, with regular user research sessions to gain feedback and help fine-tune the design and priorities.

I joined the project at its inception and was involved in the discovery of user needs and created prototypes to better understand what approach the service should take. The early iterations of the platform were developed purely in Node JS. Later versions split the backend into Django Rest Framework/Postgres/Elastic Search so a dedicated team of back-end developers could take over that part.

At one point the application used React JS with client and server side rendering though I moved back to Node/Nunjucks as at that time as there was no real benefit to the user.

The Node JS layer is a large complex codebase using ES6 with styles written in SASS, based loosely on the GOV UK style library with a number of components built on top of it. Tests are split into unit tests written in Mocha/Nock/Sinon and acceptance/integration tests written in Webdriver IO. The project is integrated with Circle CI and Heroku so tests are run automatically and a copy of the platform is deployed for each pull request so it can be demoed and tested before being merged into the live service.

Most recently I've introduced Vue JS to the codebase to improve some user interactions and make some pages easier to use in response to user feedback.

Full Stack Developer - Ministry of Justice

Mar 2015 - Apr 2016

I worked at the Ministry of Justice in the Office of the Public Guardian developing a 'Digital deputy' service. The service is designed for members of the public responsible for the finances and welfare of people who have lost the mental capacity to manage their own affairs or make important decisions. My primary role was to develop the frontend of the service inline with styles and guidelines laid down by GDS and work with the GDS community.

The service used PHP and Symfony 2 for its server-side framework and the frontend used TWIG to generate HTML pages, with styles authored in SASS and extended the GDS Elements project. The site is accessible for users of differing levels of technical and physical ability and works on all devices, ranging from people with very old computers to the latest mobile devices. The site uses progressive enhancement to improve the experience for devices capable of supporting it, including features such as autosave and real-time validation. The focus on the site is to make it as easy to use and as clear as possible. As part of the project, I extracted the controls created for it and started to develop a standalone project to make them reusable.

Javascript components were developed following TDD with tests written in Mocha JS, Integration testing was developed in BEHAT with cross-browser testing automated and integrated with Browserstack. The service was developed and deployed using Docker containers to make it easy to develop the platform anywhere and provide predictable deployment.

The project was developed following the GDS principles for service delivery, being user research driven and created by a small tight-knit team. I worked on the project from early in the Alpha stage and ultimately through a public Beta assessment and learned a lot from that.
